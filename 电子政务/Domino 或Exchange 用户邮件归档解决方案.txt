Domino 或Exchange 用户邮件归档解决方案
业务问题描述：电子邮件所管理的信息每天在跳跃式地增长，电子邮件和业务文档每天在成千上万在产生，并且占据了用户和Domino 服务器的很多空间。电子邮件正在成为业务交易的媒介，并且这些交易记录必须作为公司资产不能只是由每个人来保管。这样直接影响了Lotus Notes和MS Exchange应用的性能，也加大了客户计算机部门的负担。
一个有效的归档解决方案来将这些旧的数据从 Notes数据库中卸载下来是被期望的，同时用户能随时随地，甚至通过互联网来临时访问。此外，用户也能访问存储在内容归档系统中的数据，从而提供一个安全的、单一访问入口来满足用户的需求。
客户需求表现?
· 你们现在有Lotus Domino或MS Exchange电子邮件系统吗?
· 每个员工的电子邮件大小是不是在不停在增长?
· 邮件附件是不是占据了超过60%的存储空间?
· 你们现在有没有将生产数据库中的文档进行归档?
· 你们希望通过减小电子邮件数据库的大小来提高性能和控制存储空间以及增长速度?
解决方案描述：
IBM Content Manager CommonStore for Lotus Domino and MS Exchange 将这些信息系统和IBM领先的归档系统联系起来，包括IBM Content Manager和Tivoli Storage Manager, 这两个系统已经存在很多年了。你能从邮件系统中卸载这些文档和电子邮件，并将这些邮件和大量的邮件附件进行安全的长期的在线归档。用户能通过他们自己的邮件系统客户端随时容易地再次得到它们；更进一步，CommonStore使得用户能够恢复保存在Content Manager中的各种公司数据和业务文档，包括影像、PC文件、计算机输出报表、标准的办公文档和传真。
带来的好处：
1. 使得用户能实施一个强大的内容管理系统并且无缝地和电子邮件系统相结合。
2. 通过Lotus Notes系统直接有效地归档文档和邮件附件。
3. 减少用户端和服务器端的存储需求，并且提高电子邮件服务器的整体性能，同时给系统管理员一套工具去减小数据库的大小而不需要员工去花费很多时间去清除他们的电子邮件。
4. 使得用户能访问存放在Content Manager中的任何种类的数据来更好地运用公司信息，并且基于电子邮件系统整合公司的完整的内容。
5. 使得公司能按照自动的规则来提取和存储包含业务交易信息的电子邮件，并提供保护。
典型应用：
· 整合内容管理：客户需要一套能和电子邮件系统无缝集成的企业内容管理系统。
· Domino归档：客户需要将大数据量的Domino应用的文档归档到外部存储库中。
· E-mail 归档：客户需要去归档电子邮件。
· Combined SAP/Domino Archiving：客户需要能同时解决SAP和Domino系统的归档需求。
音频视频流管理解决方案
业务问题描述：
管理视频和音频通常是一个劳动强度很大的过程，并且基本上没有提供电子检索系统来定位合适的视频和音频信息，模拟磁带通常存放在仓库中，找起来很不容易并且异地用户必须提出申请然后等待邮寄，同时也基本不可能同步广播到多点。数字化管理其实也就意味着多点广播以及使得这些信息能在任何时间、任何地点随处可用，即使是通过Internet。
客户需求表现：
* 你现在有保存视频或音频磁带吗?这些保存的磁带用来做什么呢?
* 你希望培训用的视频能在线使用吗?
* 你希望现有的应用通过增加视频来提高价值吗?
* 你们高层领导希望实时广播信息给经理们吗?
解决方案描述：
VideoCharger使得视频和音频信息变得更加有效，可广泛用于培训、销售和沟通，它使得你能非常容易地通过网络将视频和音频信息传递给Internet或Intranet上的多个用户，视频和音频既可以实时播放，也可以先下载再播放。
带来的好处：
1) 建立一套视音频资料库来使得我们可以在任何时间、任何地点对这些资料进行访问;
2) 提供远程教育解决方案，减少出差;
3) 通过广播的方式来传递重要的信息给员工、代理商等等。
典型应用：     * 需要使用流式课件的培训教育;     * 公司信息传递 (CEO对所有经理的讲话);     * 公司媒体资产信息库;     * 使用流媒体来扩展前端应用功能。        成功案例：    1. 四川天府热线    2. 上海图书馆        案例介绍：    四川宽带网(四川天府热线)平台VOD视频点播系统        四川宽带网平台项目包含电子商务系统、视频点播会议、网上教育系统等多媒体应用系统，其中包含了大量的内容信息，因此需要一个完整的内容解决方案来管理整个平台的内容信息。IBM 公司依据系统需求以及结合IBM公司的内容管理解决方案的优势和特点为四川宽带网提供了全面的解决方案。
应用系统结构
VOD视频点播系统的硬件平台采用IBM RS/ 6000服务器，IBM视频服务器 VideoCharger 服务器 是一个带有播放影像和声音数据能力的HTTP服务器。服务器 和客户机借助 HTTP 协议进行通信。HTTP 服务器可以是任何 基于 NT或AIX平台的HTTP 服务器。
VideoCharger 可以从单 个系统扩展到多个系统以支持多个影像流。另外，VideoCharger 服务器中的组件(Application Server, Control Server, Data Exporter等)可以按客户的意愿在不同情况下 进行不同组合。这种灵活性使用户可以从小的系统开始逐渐扩 展到大系统以满足增加的需求。在VideoCharger 的所有组件 中固有的特性是提供高质量的声音和影像播放。在该视频服务器中使用了一个高性能视频文件系统(多媒体文件系统)，它可 以提供对持续时间媒体播放的实时支持，并允许多用户同时访问同一个视频文件。服务器包括网络接口模块(Data Exporter)，可以控制数据播放的速率。客户机端的 Video Charger Player for Windows95 被包括在 Video Server 包 中。
IBM CM VideoCharger Server for AIX 可根据客户机不同的连接速度灵活地以不同的速率实时地传送图象。这一产品可同时传送以下任一数据流组合： High Bit Rate,Mid-bind,Low Bit Rate 图象/声音(包 括： MPEG-1，MPEG-2，H.263 video, 和G.723 声音);Low Bit Rate(LBR) 低速率声音。这就使得每个 客户都可通过自己的网络环境连接到Video Charger Server 上，并通过一个WEB浏览器回放视频资源。
