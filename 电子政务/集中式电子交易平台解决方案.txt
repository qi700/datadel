集中式电子交易平台解决方案
1、 系统概述
企业应用平台的发展经历了几个主要阶段：主机终端模式（ Mainframe Computing ）、客户 / 服务器模式（ Client/Se
rver ，其中包括 3/N-Tier 模式）、 Web 平台模式（ Web App ）。目前 Web 平台模式已从早期的 HTTP 请求发展为基于 XML 的 Web 服务。
伴随着企业应用平台的发展变化，软件开发技术也发生了根本的变化，从面向过程的模式演变为运用面向对象 （ Object Oriented ） 的建模技术和基于组件 （ Component Based ） 的开发技术， 注重产品的技术架构（ Technical Infrastructure ）的建立， 确保系统模型的精确性和可用性，增强系统的可扩展性和可维护性。
“集中式电子交易平台”针对基础软硬件平台的特点，而又不受制于基础软硬件平台的局限，在满足原有交易业务需求的同时，可以定制和设计出更为广泛的业务应用空间。

3、 系统特点
•  交易平台的通用性： 通过提炼各个行业具有共性的交易处理流程与交易管理模式， 具有一定的行业覆盖性。能够满足银行、证券、基金、电信、商业、石油、交通等不同行业交易系统的需求。
•  交易平台的封装性：通过定制功能组件模块的方式，满足不同行业的业务特点。
•  交易平台的二次开发性：交易平台提供良好的二次开发特性。
•  基于 XML的数据交换平台：以开放的XML标准为基础，研究适合中国国情及行业应用规范的数据格式定义、数据传输控制、数据转换解析、数据存储标识等技术，为 相关领域的电子化、信息化建设提供强大的基础支撑环境。
•  基于脚本驱动的业务逻辑执行：通过采用快速软件开发技术（高级脚本语言定义与配置）实现交易逻辑控制、业务负载平衡，满足不同行业的业务需求。
