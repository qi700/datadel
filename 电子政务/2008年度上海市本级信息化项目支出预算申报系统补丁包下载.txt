第一步：安装数据库环境数据库环境：使用MSDE2000。MSDE2000的安装：打开下发光盘中的MSDE2000a文件夹。 运行setup.exe后等待进度条结束后，便完成了数据库环境的安装。第二步：系统安装系统安装：插入系统安装光盘，运行setup.exe文件，系统会自动检测是否已经安装过Microsoft.NET Framework1.1软件包，如果没有安装过将会弹出是否安装Microsoft.NET Framework1.1对话框，选择【是】。安装完成后会弹出是否安装Microsoft.NET Framework1.1语言包对话框，选择【是】。如果已经安装过相关的Microsoft.NET Framework软件则不会弹出要求安装对话框；           弹出系统安装对话框，选择【下一步】，选择系统安装路径后，选择【下一步】，文件复制完成后系统会自动检测数据库环境是否启动，如果没有启动将会弹出提示框，要求启动数据库，选择【启动数据库】，之后将会弹出数据初始化界面，选择“清空数据库”点击【开始】，完成后点击【关闭】。弹出系统安装完成对话框，点击【完成】，系统安装完成。第三步：系统首次登陆系统安装完成后，双击桌面系统快捷方式运行系统，默认用户名：admin（小写字母）； 默认密码：admin（小写字母）。首次登录系统后必须进行资料盘的导入方可以进行今后的预算申报填写工作。接收资料盘：点击 按钮，找到资料盘所在路径，点击【导入】按钮，导入完成后点击【确定】按钮后系统将会自动退出，请重新打开系统登录即可。第四步：申报填写根据单位实际情况填写申报项目。详细的填写方式可以参看《系统使用手册》。

F A Q一． MSDE出现“配置服务器失败，请参考系统日志说明”和安装过程进度条退回。           原因：系统配置不符合Microsoft SQL Server Desktop Engine安装要求。 解决办法：MSDE 2000 要求安装 Microsoft Internet Explorer 5.0 或更高版本。最小安装便已足够，Internet Explorer 不必是默认浏览器。 （1）控制面板”中，双击“网络连接”。 （2）在“高级”菜单中，单击“高级设置”。 （3）在“适配器和绑定”选项卡上，确定选中了“Microsoft网络的文件和打印机共享”。 二． MSDE 2000 安装失败。如果下列任一安全策略已被设置为“禁止安装”，则 MSDE 2000 安装将失败： ●Windows XP 的本地安全策略“设备：未签名驱动程序的安装操作”。 ●Windows 2000 的本地安全策略“未签名非驱动程序的安装 操作”。 如果使用了“禁止安装”设置，则必须在安装 MSDE 2000 之前将该设置更改为“默认继续”。如有必要，可以在完成安装之后将该策略还原为以前的设置。 要设置这些策略，请执行下列操作： 1、在“控制面板”中，双击“管理工具”。 2、双击“本地安全策略”。 3、展开“本地策略”。 选中“安全选项”。 确保在安装 MSDE 2000 之前，右窗格中的下列选项被设置为“默认继续”： 对于 Windows 2000：“未签名非驱动程序的安装操作”。 重新安装前将已经安装的sqlserver目录删除。 三． 出现“指定的实例名无效”。          原因：将SQL Server没有从添加删除程序中删除。并将已经安装的sqlserver目录删除。 也有可能你的某项服务不能启动，导致安装失！在重新安装前配置服务一下服务！解决办法：
