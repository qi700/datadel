集中备份 备特佳提升乐山市政府信息化
随着社会的发展和科技的进步，各种业务的运营越来越依赖于信息数据的完整和IT系统的稳定运行。政府的信息系统更是如此，因为政府的信息数据与国计民生息息相关，甚至和社会稳定密切相关，出不得半点纰漏，必须做好容灾备份。而四川省乐山市信息中心采用和力记易的备特佳系统，实现了不同操作系统、不同数据库类型、不同数据库版本之间数据的异地集中备份，而成为政府行业的成功经典案例。
一、 政府行业必须做好容灾备份和业务连续性管理
政府办公及业务运营越来越依赖于各项信息系统，各种政务档案资料都几乎全以各种形式的数据存放于相关的信息系统中，但是这些重要数据同样处在一个充满风险和威胁的世界里：无法预知的IT硬件设备的损坏、断电、火灾、自然灾害、恐怖袭击等，造成数据丢失或业务的突然中断;系统人员误操作造成意外宕机或关键数据丢失;手段频多的黑客攻击、病毒入侵、垃圾邮件、网络与系统的漏洞，造成网络瘫痪、系统崩溃等。而如果不能对风险采取有效治理，一旦数据由于上述某种原因丢失，不仅可能会造成整个运营停滞，造成巨大经济损失，更会影响到公众的正常生活 甚至是社会的正常运行。
另据一项调研数据显示，近年来，政府行业信息化建设的重点方向主要是内网建设、电子政务业务系统、办公自动化(OA)、外网建设、数据库建设和信息安全，而信息安全其实就包括了数据容灾备份及业务连续性管理。
因此，保证政府单位的数据安全以及业务的连续性，已经成为政府行业单位信息化建设的重中之重。
二、 环境多样 解决方案需做到整体兼顾
本案例中四川省乐山市信息中心集中了全市门户网站、信息公开、行政审批、交换平台、财政核算、劳动市场信息、社保网等部门的所有重要数据。这里是一个数据非常集中的地方，也是政府重要部门工作的平台(如图1)。
▲图1 四川省乐山市信息中心
这些部门采用了不同的操作系统，包括Microsoft Windows Server、Sun Solaris(UNIX)、IBM AIX(UNIX)等，而且在这些操作系统上运行的数据库也不尽相同，既有Microsoft SQL Server，又有Oracle，同时数据库版本也不一样，有Oracle 9.2，还有Oracle 10g，要实现异地集中备份，难度较大，解决方案需做到整体兼顾。
而且，乐山市信息办对于数据安全尤其重视，拟在异地建立异地中心机房，将本地机房的各项重要数据实时备份至异地机房，以保证重要数据的安全。只是囿于操作系统和数据库的类型、版本太多，预算费用过于高昂，所以虽有建立异地灾备中心的想法，但一直没有付诸实施。
三、异地集中备份 备特佳提供完美解决方案
乐山市信息中心得知北京和力记易科技有限公司(以下简称我司)拥有容灾备份解决方案，力邀我司演示技术方案，并随即在财政核算(使用的是Sun Solaris操作系统)的小型机上购买了一套“备特佳”，以作为对我司技术实力的考查。我司工作人员在第一套Sun Solaris方案交付使用后，随即推出第二期整体解决方案(如图2)。
图2 乐山市信息中心备特佳解决方案拓扑图
如上图，“生产中心(A)”环境如下：社保网群集系统包括Server1(工商系统)、Server2(失业保险)、Server3(养老保险)、Server4(医疗保险)，采用的是IBM AIX 5.3，数据库为Oracle 9.2;Server5为劳动市场信息系统，Server6为财政核算系统，这两台服务器操作系统为Sun Solaris v890，数据库为Oracle 10g;Server7为交换平台系统，Server8为行政审批系统，Server9为信息公开系统，Server10为门户网站系统，这4台服务均运行Microsoft Windows Server操作系统，使用Microsoft SQL Server数据库。“备份中心(B)” 是异地容灾备份中心，地理位置设在乐山市所辖的井研县机要局信息中心，其环境如下：BackupServer1-BackupServer4均安装Microsoft Windows Server操作系统，其中BackupServer1用于对社保网群集系统的四台服务器Server1、Server2、Server3、Server4做集中备份;BackupServer2用于对劳动市场信息系统Server5做备份;BackupServer3用于对财政核算系统Server6做备份;BackupServer4则用于对交换平台Server7、行政审批Server8、信息公开Server9、门户网站Server10四台服务器做集中备份。
具体部署如下：1、在“生产中心(A)”的Server1-Server10上部署备特佳主站，其中包括Windows版、UNIX版，同时在“备份中心(B)”(异地)的BackupServer1-BackupServer上部署备特佳从站;2、将“生产中心(A)”Server1-Server10所产生的数据通过备特佳实时集中备份到“备份中心(B)”(异地)对应的备份服务器中，RPO=0，确保数据零丢失;3、当“生产中心(A)”任意一台或多台服务器因为磁盘故障或者系统故障导致业数据库能正常使用时，可以从“备份中心(B)”(异地)相应的备份服务器上进行数据恢复，从而使业务系统恢复正常;4、当“生产中心(A)”服务器的数据因为出现逻辑性错误(操作失误，误删除等)导致数据错误或者丢失后，数据库可以从“备份中心(B)”(异地)备份服务器的回退数据进行任意时间点的数据回退，且保证回退点数据100%完整可用。
四、最少钱办最好事 客户满意度提升
此解决方案，连同硬件，乐山市信息中心只花费了100多万元，相对于以前只敢想不敢做，到现在以最少的投入建立异地灾备中心，信息办主任乐呵呵地告诉我司工作人员：“我们这次是花了最少的钱，办了最好的事。信息中心的数据安全一直是我们的一块心病，以前数据只备份在本地，而且还是定期备份的，晚上觉都睡不好啊。最怕晚上有电话打过来。而现在我们做到了实时将数据实时备份到异地，哪怕是机房发生了火灾，我们也不怕数据丢失了。我终于可以睡个安稳觉了。”
