自动备份 让你远离误删数据噩梦
自动备份程序 　　自动备份程序，不管是用于创建本地备份还是通过高速的互联网连接复制数据，都能够简化管理任务。正确配置的自动备份程序，包括Remote Data、Backups、Spare Backup、Dr Backup、Yosemite Backup、Windows NT Backup、Symantec/Veritas Backup Exec等，能够减少管理员的工作负担。 　　通过使用自动备份程序，可以清除原来那种手动备份一个单位关键数据的压力，从而为管理人员节省了大量时间。然而，在自动备份程序使用过程中，我们有可能变得过于自信。例如，有时你会发现，由于一个技术人员误删除了硬盘上的数据，你满怀信心地试图从一个备份磁带上恢复数据，结果却发现磁带不可读！如此一来恢复成本变得高昂起来。 　　不过，如果检查如下关于自动备份程序的“十项注意”，可以避免使我们的工作进入这种噩梦之中。 　　磁带并非值得绝对依赖 　　这真是一个可悲的现实。许多昂贵的磁带系统在最需要的时候失效，更糟的是，许多磁带失效并没有及时被发现。不管是由于磁带驱动器没有及时清洗造成的，还是由于时间的原因造成的介质故障，─通常情况下，磁带系统故障只是在最关键的时候才被发现。如果发生了这种事情，您可能要花上大量时间重新建立工资或人员数据库了。 　　磁带维护的风险性 　　除了磁带驱动器和磁带自身存在着各种问题，操作人员能否正确操作媒体也是一个问题。如果管理员和其他人员不能用正确的媒体完成这项任务，磁带备份有可能毫无价值。即使老练的IT专业人士偶尔也会将某一天的磁带备份用于另外一天的工作上。正因为这个原因，认真监视、跟踪时间和媒体的对照关系是至关重要的。 　　数据位置改变 　　数据的位置随着时间的推移而转移和改变。例如，一个组织与公众的联系文件原先可能存储在服务器的名为“public”的文件夹中，可是在日后的数据收集过程中，企业采取了新的存储策略，转而将同样的“public”文件夹及其文档转移到一个名为“Marketing”的文件夹中。这种事情还会发生在数据库、电子邮件账户、用户目录、部门文档以及其它数据上。如果备份操作不能随着数据存储位置的改变而更新，备份就面临着丢失关键数据的危险。 　　备份操作偶尔失效 　　仅仅备份操作按照计划完成并不意味着备份程序就正常完成了。因为，有时候会停电，发生雷电，备份介质容量不足，备份驱动器变脏，系统死机等等。这些因素会使备份无休止地进行下去。因此，你千万不要因为备份操作已被安排和计划就认为它是万能的。相反地，你应该把检查备份日志作为一个每天的例行程序。还要使恢复备份成为一个测试有效性的手段。 　　备份操作也会备份错误的数据 　　在正常完成时，备份操作倾向于完全按照程序所要求的那样完成这一过程。备份操作并不在乎自从上次运行以来，整个目录或分区是否已经删除；备份程序通常会备份要求它们备份的数据。用户有时会错误地删除整个文件夹和目录，但却需要几天来认识到这个错误。如果你的组织通过单个的备份程序完成日常的操作，那么恢复误删除的数据的可能性就会日益减少。维持多个备份集（或者在一周之内执行几种不同的备份）就会给企业恢复数据提供额外的选择。 　　数据库、Exchange需要TLC 　　许多应用程序─包括那些依靠Microsoft SQL Server和Microsoft SQL Server Desktop Engine（MSDE）来管理其数据的程序，都将其关键性的信息存储在多个数据库文件中。除非将在那些数据库之间链接信息的复杂指令也进行了备份，仅仅将数据库文件保存到一个备份驱动器上并不能确保成功的恢复。在使用这种第三方软件时，一定要遵循制造厂商的备份指南。 　　Exchange服务器也需要特别对待。电子邮件服务器需要能够在线备份的应用程序，因为设想一个组织仅仅为了备份操作而关闭其电子服务器是不切实际的。与此正相反，一个组织必须要保证其备份程序支持在线的或活动的操作。就Microsoft的流行的电子邮件服务器而言，这种程序被描述为能够发现Exchange的存在。 　　某些应用程序要比其它的工作良好 　　许多厂商承诺了许多甜言蜜语，不过其产品并非都如所描述的那样性能优良。一些应用程序不能备份你指定的所有文件、文件夹和驱动器。另外有一些程序，即使你要求增量备份，它们也是执行差异备份。还有一些软件并不能将数据正确地写入特定的媒体或者并不能在合理的时间帧内完成任务。 　　更糟的是，对在线备份空间的竞争导致了许多供应商处于歇业的状态。通常情况下，公司会在不知不觉间破产，而且将你的数据也会带走。因此，在考虑软件厂商和在线供应商时，一定要仔细选购。在选择一个备份合伙人时，名声和可靠性要超过成本的节约。不管什么时候，只要可能，在将一个应用程序部署到一个生产性环境中之前，首先对其进行测试是也一个最佳的方法。这样做有助于在破坏发生之前，首先将一些不正常的和不兼容的问题揭示出来。 　　编制文档很关键 　　防止数据丢失以及任何灾难恢复规划的关键要素是文档编制。只有通过记载哪些系统备份了哪些数据，一个组织才能满怀信心地认为自己对其数据进行了恰如其分的保护。除了跟踪这种信息，文档编制还应该为测试备份提供指导，以确保备份能够正确地恢复。 　　恰当的备份策略需要经常性的检查 　　一个企业中的数据位置是经常改变的。通常，文档编制并不能跟上改变的步伐。这样一来，组织的备份系统跟踪错误的数据成为就成为家常便饭。IT部门能够通过经常检查其备份策略来防止灾难的发生。一年四季经常检查备份策略可以跟上企业改变的步伐。 　　安全性容易被忽视 　　一旦进行了数据备份并不意味着就万事大吉了，还需要考虑安全性。如果因为备份被放错地方而造成失窃，将造成更大的潜在威胁。因为备份通常包含着至关重要的和受保护的信息，公司不但要尽力保护重要的数据，而且还要保护备份。 　　事实上，根据企业所属的行业，需要采取特别的措施以保护备份被公诸于众。一定要保证在扩展备份和恢复的特权时，在处理备份媒体时，采取恰当的措施预防未授权的访问。对于在线备份来说，这意味着供应商支持128位的加密数据流以及用于恢复的单独加密密钥。
