国家电子文件支撑平台系统方案
项目背景：
中华人民共和国国家档案局，是中华人民共和国国务院的直属机构之一，主要工作职能包括掌管全国档案事业。国家档案局对全国档案事业实行统筹规划，组织协调，统一制度，监督和指导。集中统一管理党和国家中央机关的重要档案资料，保守党和国家机密，维护档案的完整，确保档案资料的安全。负责接收、征集、整理、保管党和国家中央机关的重要档案资料，推进档案工作的科学化管理和现代化建设，并制定档案工作人员队伍建设规划，组织档案专业教育和档案专业干部培训工作等。
随着信息化的深入发展，电子计算机在党和政府工作中得到广泛应用，电子文件的长久保存、集中管理和有效利用正成为国家档案工作的重要组成部分。在当前国家信息化快速发展，电子政务扎实推进，电子文件、电子档案大量产生的形势下，对电子文件管理实行规范化、法制化，建立有中国特色的电子文件管理体系具有重大意义。将有利于理顺电子文件管理体制、机制上的各种关系，有利于我们从国家层面、法规层面、服务层面上去同有关部门共同规划和推动电子文件管理工作。
客户需求及业务挑战：
中华人民共和国国家档案局国家电子文件支撑平台系统项目建设在认真总结我国十多年电子政务建设正反两方面经验与教训，总结民族软件企业发展模式的利弊的基础上，认真梳理、归纳和分析党政机关、企事单位对电子文件管理的共性业务与技术需求。在此基础上研究、编写电子文件全程管理标准规范体系，本项目严格遵循统一标准研发《国家电子文件系统支撑平台系统》所需的：“共性应用基础支撑软件”、“通用工具支撑软件”、“共性业务与通用功能组件”、“共性办公应用系统”和“系列标准规范”。最终形成符合我国国情的、具有自主创新性与示范意义的、能够被社会所认可的、公开标准的、并可持续发展的国家电子文件支撑平台系统。初步实现我国对电子文件系统按统一标准规划、设计、建设、应用与管理，推动我国电子政务的标准化、国产化进程。
在本项目的建设过程中，非常关注数据库产品的选择，人大金仓通过与国家档案局的多次交流，深度了解了本项目的建设需求，提供了优秀的KingbaseES大型关系型数据库产品，为支撑平台的顺利建设提供了有力保证。
方案部署及特点：
在本项目的建设中，KingbaseES充分体现了如下的突出特点：
数据库有良好的兼容性，能够兼容MS SQL Server数据库的主要特性，需要对前期基于MS SQL Server 上开发系统进行移植，必须保证系统移植的正确性和完整性；
产品除了能支持除了常见架构服务器，还需支持国产龙芯CPU机器，支持主流Linux操作系统；
产品安装与使用界面易理解、易浏览、可操作性强，易学易用；
产品界面友好，可视化程度高，操作简便，易学易用，便于维护和持续开发；
产品具有强大完备的图形化管理工具，方便对数据库进行管理；
系统需具有TB级以上的数据处理能力，面对大数据量数据能有良好的处理能力；
具有良好的可扩展性，可满足业务数据量增大时的系统扩展要求；

应用效果：
标准性
系统的建设为电子文件管理构建了统一标准化的平台，给用户能够高效地在该平台中应用到日常电子文件管理工作。
实用性
提供了一个标准使用的用户平台，用户的常用功能电子文件管理、文件管理标准化、档案保护等功能的轻松使用，较大的提高了业务操作效率。
简便性
提供友好的人机界面、模块清晰、操作方便快捷，不仅应用操作简便，对于系统管理员，也能够通过数据库管理工具，轻松便捷地对数据库进行日常维护操作。
安全性
电子文件涉及到很多相关重要文件，所以对系统安全级别很高，金仓数据库的安全级别（公安部三级认证，相对于国际认证B1级）比国外数据库在我国销售的数据库版本安全级别（最高不超过C2级）有更高优势，在数据库级增强了整个系统的安全性。
客户评价：
金仓数据库在项目的成功应用，体现了国产数据库KingbaseES有着很高通用性、兼容性、可靠性、安全性。在使用管理过程中轻松方便，系统至今运行良好，证明了在实际项目系统中使用金仓数据库KingbaseES的完全可行性。相对于国外数据库，在付出较少成本的情况下，项目得到了更多的优质服务。
商业价值：
电子文件是信息社会中党务、政务运行的一个重要手段，是信息化条件下文件和档案的一种新型载体，是我国档案信息资源的重要组成部分。国家电子文件支撑平台系统的成功上线，是创新电子政务相关全套软件按工业化模式系列化、产品化、标准化和社会化方式的建设，落实党中央在“十六大”、“十七大”报告中提出“两化融合”的战略要求；创新各类综合办公形成的电子文件办公业务与电子档案业务、信息系统共性应用软件技术、管理模式与标准体系一体化建设模式，较大提高了电子文件的标准化管理水平。
金仓数据库KingbaseES成功应用于创新电子文件相关软件，优质良好的通用性，兼容性，体现在多种硬件服务器及操作系统上安装部署及支持多种实用通用的功能，并且通过提供原厂的技术支持服务，利用强大的迁移工具KingbaseES JDTS，很好地将前期基于MS SQL Server开发的应用系统移植到金仓数据库KingbaseES上，并且本地原厂的技术支持服务的快速响应，使项目过程中得到了数据库技术方面得到了有力支持，保证了系统的成功上线运行。
