Exchange Server 构建政务信息方案
朝阳区是北京最大的一个区，其中包括奥运体育场馆、CBD、使馆区等很有影响的单位和企业都在这个区内。朝阳区信息网络中心成立于2001年底，是 隶属于朝阳区信息化工作办公室的行政事业单位，主要负责朝阳区的日常维护和软件开发，以及区内重大信息化工程项目的指导和监理工作。朝阳区是北京最大的一个区，其中包括奥运体育场馆、CBD、使馆区等很有影响的单位和企业都在这个区内。朝阳区信息网络中心成立于2001年底，是 隶属于朝阳区信息化工作办公室的行政事业单位，主要负责朝阳区的日常维护和软件开发，以及区内重大信息化工程项目的指导和监理工作。
为使朝阳区信息化建设在技术上、应用上达到国际一流水平，朝阳区信息化工作办公室与微软(中国)有限公司就"数字朝阳"的建设开展了紧密的合作, 目标就是要逐步实现以电子政务、城市管理和社会信息化为核心，全面提高政府的管理能力和服务水平；依托电子政务手段，推动政府管理模式和体制的创新，促进 政府部门资源共享和政务公开；创建法治型、服务型、节约型政府。
在针对各职能部门之间的信息沟通交流、政务事件的处理响应速度、信息及时获取和发布等问题上，朝阳区政府对政务内部办公信息化建设提出了快速建立一 套内部信息交流平台，以使信息交流渠道畅通；同时要保障信息交流平台的稳定性和可靠性，以满足政府部门对信息化的依赖；并且要充分利用现有技术人员和设 备，投入建设资金尽可能少的要求。
为此，朝阳区政府信息中心在认真评估了Microsoft Exchange Server 2010任何地点的访问、保护和遵守法规以及灵活可靠这三大商业价值后，参加了微软Exchange先行体验计划，将Microsoft Exchange Server 2007升级到Exchange Server 2010。
方案背景
随着经济全球化、社会信息化的不断发展，人们的工作方式和生活方式也在不断改变。面对这种改变，作为管理国家事务和社会事务的政府组织，其唯一选择 是顺应时代要求，运用网络及相关信息技术，改进工作流程和服务手段，带动体制改革和管理创新，提升内部管理水平，提高为社会组织和公民服务的质量。因此， 一场信息化革命正在中国政府机构中悄然兴起。朝阳区作为北京市最大的人口和经济区县，其政府信息化建设一直走在前列。
作为朝阳区政府信息化建设的一部分，其政务内部办公信息化一直具有举足轻重的地位。在实施信息化办公以前，朝阳区政府的日常办公存在诸多不便。例 如，各部门及其下属各部门之间的信息交流不畅通，严重影响了政府处理各种事件的响应速度；政府职员缺乏一个统一的信息交流平台，导致很多信息不能及时获取 和发布。针对这些问题，朝阳区政府对政务内部办公信息化建设提出了以下要求：
快速建立一套内部信息交流平台，以使信息交流渠道畅通；要保障信息交流平台的稳定性和可靠性，以满足政府对信息化的依赖；充分利用现有技术人员和设备，投入建设资金尽可能少，以符合政府部门"少花钱多办事"的原则。
基于以上考虑，为了尽快实施内部办公信息化并满足朝阳区政府提出的三项要求，朝阳区信息网络中心决定采用微软的系列产品和解决方案作为内部办公应用的主体架构。
解决方案
在全新规划的解决方案中，系统服务器端运行在Microsoft Windows Server 2008操作系统平台上，利用操作系统集成的IIS来提供Web服务。同时数据库采用Microsoft SQL Server 2005，用来存放各种类型的办公信息；邮件服务器采用Microsoft Exchange Server 2010。客户端运行Windows桌面并通过IE Web浏览器经区政府内部网络与服务器端进行数据交换。
在升级后的Microsoft Exchange Server 2010平台上，用户不但可以通过Outlook连接到全新的Exchange Server 2010服务器，还能够通过最新的OWA Web访问界面实现电子邮件的在线收发。通过Exchange Server 2010的高可用解决方案，实现了更为高效、低成本的高可用，保障邮件系统的实时在线，为朝阳区政府邮件系统用户提供了更高的服务质量保证。
为了能更好地实现Microsoft Exchange Server 2010全新的高可用性，在该方案中通过虚拟化技术把Microsoft Exchange Server 2010搭建成为一个具有高可用性的服务器群集，即虚拟出两台服务器来装Exchange Server 2010，这两台Exchange Server 2010在数据库级别互为备份，即使一台服务器上的数据库不可用，另外一台Exchange Server 2010服务器会继续不间断的提供服务。
除了存储和可用性功能的变化之外，Exchange Server 2010引入了处理高可用性更简单的方法，特别是在Windows Server 2008环境下。Exchange Server 2010大大简化了群集邮箱服务器的全部处理过程，并把简化过程提高到了一个新水平。Exchange管理员不必考虑底层群集组件的配置问题，因为这些工 作由负责Windows群集管理的Exchange工具在管理员视野范围之外来完成。
数据库可用性组可以支持多达一个数据库的多达16个副本，因此用户不必担心数据丢失。此外，由于在Exchange 2007中群集邮箱角色不能与其他角色并存，而现在，其他的服务器角色可以与高可用性邮箱服务器并存，因此减少了Exchange的又一整体硬件要求。
Exchange Server 2010做出的最大改进其中之一就是减少I/O占用系统空间。这种改善很大程度上归因于Exchange 2010完全依赖64位架构，这种架构随之也带来很多好处，如增强的RAM支持。Exchange 2010把这种I/O调用的降低提高到一个新水平，进一步把部分页面大小增加到32K。凭借此及其他方面的改进Exchange 2010的I/O需求比Exchange 2007减少了50%到70%。这就意味着单台Exchange 2010服务器可以支持比Exchange 2007系统更多的邮箱。同时， Exchange 2010还提供更大的存储支持，存储邮件时减少了SATA硬盘的用量。
优势和收益
朝阳区政府内部信息平台的建设，顺应了时代发展的要求，改善了政府职能部门的服务水平，提高了政府职员办公效率。与此同时，选择微软产品和解决方案 为客户带来了巨大收益。虽然 Microsoft Exchange Server 2010 的评估过程仍在进行中，但朝阳区政府对最初的结果非常满意，而且考虑在不久的将来在整个范围内部署该解决方案。
降低了邮件平台的管理和部署成本，提高了可用性
Microsoft Exchange Server 2010可提供灵活可靠的高可用解决方案，政府信息管理员不需要对Windows群集技术有特别深入的了解，就可以很灵活、简便地部署Microsoft Exchange Server 2010高可用组DAG，而且不需要共享存储设备的支持，大大的降低了存储成本。朝阳区政府采用两台服务器来部署Exchange Server 2010的高可用解决方案，这比之前可能至少需要4台服务器的方案结构节省了一半的硬件成本。
使电子信息更遵循政府信息存档和法规的要求
Microsoft Exchange Server 2010提供了一种全新存档功能，与其他增强的邮箱管理功能相结合，提供了包括多邮箱高级搜索，法律保持和精细保留策略功能。将这些功能与 Exchange存储体系相结合，可以改进对信息的控制，同时为用户提供流畅的体验，保持一致的日常性管理收件箱的方式。
另外考虑到政府职能部门的特殊需要，通过存档进行有效的管理分类，查询，直接保存用户主邮箱和个人存档中用户已经删除和编辑过的邮箱项目（电子邮件，日程，任务等）。这些存档资料将成为政府法律发现和其他与法规遵守相关的调查中的主要数据来源。
北京市朝阳区信息网络中心冯骏主任在谈起该系统的用户体验时提到："我们通过信息上报采编系统可以在第一时间了解每天发生在区内的各种事件和新闻， 这些信息不断及时而且全面。我们根据这些及时全面的信息可以了解到群众身边的各种困难以及他们所关心的问题，以便快速地做出决策，更好地为公众服务。而在 此之前，所有的工作都是通过电话或文件形式来传递这些信息，不仅不方便而且还没有信息存档，不便于提供决策支持。" 整个信息系统从2009年9月初开始使用，该系统每天处理的信息量达到几十条，使用至今已处理数上千条的信息。
随时随地访问信息，提高政务办公效率
Microsoft Exchange Server 2010 可以帮助员工从各种不同的客户端和设备连接到其电子邮件、语音邮件、日历和联系人。除信息等待功能外，Exchange 2010中还增加了呼叫应答规则、未接电话短信（SMS，Short Message Service）及文本信息提醒功能、语音邮件通知功能，以及语音邮件文本预览功能 "借助 Exchange Server 2010可以非常方便地访问其消息"，朝阳区信息中心的信息工程师李嵩说，"员工们可以通过 Outlook 2010、电话、或移动设备即时了解信息。"
