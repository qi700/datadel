北京市朝阳区内部办公平台方案
“北京市朝阳区内部办公平台[以下简称办公平台]”是北京市朝阳区政府在政务专网上建立的一套办公平台，整个平台包括三个子系统：朝阳区政务专网门户网站[以下简称内网]、朝阳区政务专网统一授权系统（以下简称统一授权系统）和朝阳区统一消息系统（以下简称消息系统）。
统一授权系统，将会对全区所有用户帐号进行统一管理，进行统一授权。同时对于已经开发的老系统，可以在不改变应用系统原有权限管理系统结构的前提下，将原系统权限与统一帐户做关联；正在或将来新开发的系统，则可以利用统一授权系统中提供的开发接口，直接实现应用系统权限整合。从而最终形成朝阳区政务平台应用系统用户管理及权限管理的开发标准。
借助统一消息平台，用户之间可以进行多种点对点的沟通，包括即时文字通讯、文档传输、音频及视频对话等；另外，统一消息平台可以在第一时间以多种方式通知用户有什么新信息产生，让用户不会错过重要信息；同时通过统一消息平台提供的开放接口，各单位可以对消息系统进行二次开发，扩展消息系统的功能，使之能无缝地和各单位的应用系统进行整合，实现定制通知功能。从而最终形成朝阳区政务专网统一消息通知开发标准。
内网作为朝阳区政务专网的门户网站，最主要的功能是对全区用户展现最新的通知通告、文件、电子期刊、新闻等资源，同时内网也是各个应用系统的入口，每个用户由内网通过各自帐号可以登录各个应用系统，真正实现单点登录。
1.1 朝阳区办公现状
朝阳区信息化工作办公室是全区信息化的主管部门，负责指导、组织和实施辖区内的信息化建设。目前，随着朝阳区信息化水平的不断提高，运行在政务专网上的各种应用系统逐渐增多。作为全区信息化建设整体规划以及各应用系统的管理单位，虽然能够正常完成工作，但一些弊端已经不可避免地暴露出来。
原有的内网技术陈旧，原系统在稳定性、开放性、操作性及扩展性等方面已经不能满足现有的需求。虽然系统数据库中存储了大量资源，由于原系统的搜索功能不够强大，造成很多有用信息无法检索。网站的风格已经不适合现在的需求。
各种系统的用户管理以及权限管理都是各自开发，造成用户使用多个系统必须多次登录，大大降低了工作效率，内网没有实现单点登录。
用户主要是靠登录到内网浏览网页来获取各种信息，这种方式常常会导致信息获得地不及时，耽误工作。
1.2 条件和限制
利用全区已经部署的ActiveDirectory来进行用户管理，邮件系统的整合必须利用全区已经部署的基于 Exchange2003 server建立的内部邮件系统。消息系统必须基于Microsoft Live  Communications Server 2003或者Microsoft Live  Communications Server 2005来开发。
第二章总体设计原则
2.1 系统设计原则
2.1.1 平稳移植现有资源、数据，高起点升级开发
朝阳区原内网积累了近两年来区内发生的重要新闻数据及各委办局上传的部门信息数据，对区内的信息沟通及交流做出了贡献。新的内部办公系统将充分发掘原有的数据资源，保持原有的优秀栏目及应用，同时进一步优化栏目，调整这个界面样式。
领航科技应用微软SharePoint Portal Server门户应用，将原有的内网信息统一集中到新的内网办公平台门户网站上面，应用先进的网络模糊搜索引擎，充分调用原有的信息数据为全区信息化办公决策提供全面详实的真实数据。
2.1.2 提高内网信息更新率、综合使用率；通过“分散维护、集中管理”，减轻信息办的维护工作量
随着网络应用的不断提高，信息资源的不断增加，原有内网信息更新率不高，综合使用率低的现象日益呈现出来。因为原有系统设计的局限很难将网站大量的维护工作量分解开，分解到全区几十个相关的委办局。久而久之这个网站的信息更新很难达到辅助于领导决策，很难应对区内用户对全区重要事件的浏览查询。尽管主管全区信息化建设的信息办尽了自己的努力，充分调动网管人员与采编辑人员的工作积极性，仍然很难满足网站不断求新，求细的信息量更新的要求。
领航科技基于以上问题的分析，在本项目的投标建议书里推荐采用微软强大的内容管理系统，同时集成微软的活动目录用户身份认证技术，既解决了以往网站栏目设计死板，很难及时、灵活地修改的弊病，又最大范围的将设计的各个栏目的信息发布权限下放到区内的各个委办局；同时保持了信息审核的权限，通过信息办对各委办局，各个栏目新闻的进行审核后，再行发布。这种结构既满足了全区内部办公平台的信息的权威性、政策性和针对性，同时也提升了全系统的信息更新的时效性。通过对全区内部办公网信息发布系统的改造，使内部办公平台真正成为朝阳区办事人员的内部新闻和业务信息门户，成为内部交流与沟通的门户与桥梁，成为辅助区领导决策、体察办事人员情况的窗口。同时，该门户系统最大限度地减轻了信息办的维护强度，使信息办工作人员有更多的精力设计栏目；调研全区的业务需求，产生更多更好的创意，为整个朝阳区的信息化服务。
2.2.3 提供国际标准技术平台与应用集成接口，为朝阳区内各局委办应用建设提供基础平台与技术规范
基于Windwos2003的活动目录技术，实现了整个系统的人事及权限信息以 “域树”的结构进行管理，在进行人员权限管理、认证、软件自动分发、客户端群组管理方面提供巨大技术优势，极大提高了整体系统的维护水平、降低了维护成本。联机数据分析(OLAP)与数据挖掘技术（DM）的应用，为未来多系统业务数据整合、分析和辅助角色处理，提供技术实现平台，同时也大量降低系统数据迁移的成本。
2.2.4 重视交互界面设计，提高全区新内部办公系统的易用性及实用性
原有内网设计上存在局限性，没有充分考虑维护人员与使用者；信息发布人员与信息浏览者等诸多情况下的不同用户角色差异交互。例如，时效信息发布后没有相应的评论和浏览的统计；新闻及各个栏目的管理者不能行之有效的对整个网站的趋向和热点情况做及时地统计更新等；另外，从体系结构上分析，原有内网系统用户界面层、逻辑处理层与数据交换层没有完全分离，因而增加了界面更新、栏目扩展的难度。诸如此类的问题，严重制约了该内网在全区系统内的实用性。
鉴于以上所述情况，领航科技重新规划设计了信息发布栏目，预制设计了新闻评论、浏览日志纪录、点击率和访问统计分析等功能，在保留原内网设计有点的同时，有针对性地将国际领先的新闻类站点管理模式引入朝阳区内部办公网，提升整个内部办公网的信息交互性。配合领航科技在微软Live  commucation server 2005上的应用开发，不仅保留了门户站点的时效性，同时又加强了信息的针对性。通过LCS 在朝阳内部办公网的应用，简化原有的系统的使用难度，实现了以往用户要学着如何用到相应的信息直接推送到用户桌面的“傻瓜式”转变。达到信息及时用、及时浏览的要求，让朝阳区新的内部办公平台真真切切成为朝阳公务人员工作生活的一部分。
2.2 项目工作目标
开发新“信息发布”和“通知发布”系统：优化现有信息维护功能；提升稳定性、可扩展性的系统性能
提供“统一认证平台”解决现有业务系统相互隔绝的问题
提供“统一消息平台”实现信息通知的即时、精确发送
完善领导特区功能，为领导提供高质量信息服务和辅助决策支持服务
2.3 整体解决方案
朝阳区新政务办公平台的架构将由如下几个部分组成：
2.3.1 基础网络平台
朝阳区（朝阳区政府信息办）现有网络设施可以满足内部办公平台的运行。
2.3.2 应用支持平台
朝阳区（朝阳区政府信息办）现有软件环境可以满足内部办公平台的运行。
从应用支持平台角度分析，该新政务办公平台从服务器端到客户端均将采用符合国际标准.NET架构体系的产品和技术，利用ViualStudio.Net进行研发，构建新一代朝阳区政务办公平台。
应用支持平台是朝阳区内部办公平台的最终承载平台，为朝阳区政府内部各委办局集中提供应用服务，形成全区机关统一的应用基础支撑体系。主要包括：信息门户平台、应用服务器平台、数据库服务平台，目录服务、邮件服务等，为上层业务系统提供运行以及管理维护平台。
目录服务完成电子政务中数字身份的统一管理、用户验证和安全控制等，可与安全认证中心集成。
下图是利用windows 2003中的活动目录实现的朝阳区内部办公平台的单域模型的示意图。
基于windows 2003中的活动目录为每个用户创建邮箱、LCS帐号。根据用户的属性创建安全组(Group)。
2.3.3 政务内网门户网站
根据朝阳区内部办公平台的总体需求，政务专网门户网站是一个内含多种应用系统的集成体系，利用各类应用系统建立统一的信息平台。本方案中提出采用SharePoint Server 2003作为政府专网门户的平台。因为SharePoint  Portal Server 2003 使得技术人员能够开发出智能的门户站点，该站点能够无缝连接到用户、团队和知识。 SharePoint Portal Server 2003 提供了一个整体的业务解决方案，它利用了单点登录和应用程序集成功能，以及灵活的部署选项和管理工具，将来自不同系统的信息集成到一个解决方案中。该门户站点使得端到端的合作更为容易，实现了个人、团队和信息的整合、组织和搜索。用户可以通过门户内容和布局的定制和个性化以及目标受众，更快地找到相关信息。
2.3.4 信息管理系统
根据朝阳区内部办公平台的总体需求，政务专网门户网站上的信息由下属相关部、委、办人员发布，信息办审批。本方案中提出采用 Content Management Server 2002作为信息管理系统的底层平台。因为 Content Management Server 2002提供了内容投稿和传送、网站开发及整体网站管理的完整功能组合，让各委、办、局之间能够有效地建立和部署继承政务专网、各自内部网络、甚至外部网络的网站。
2.3.5 统一认证平台
朝阳区政府专网中存在不同时期利用不同技术平台搭建的多种应用系统，各种系统的用户管理以及权限管理都是各自开发且相对独立的，造成用户使用多个系统必须进行多次登录，极大地降低了工作效率。本方案中提出使用Identity Integration  Server 2003整合内网中多个系统上的用户和权限。政务专网上的每个用户都可以直接使用AD帐号，通过统一授权系统提供基于角色的权限控制机制，在各种应用系统中漫游。各种应用程序则可以根据自身需求定义自己的权限和角色，由统一授权系统建立用户到角色及角色到权限的映射关系，提供灵活方便的访问。
2.3.6 统一消息平台
朝阳区用户主要依赖域身份登录到内网，浏览网页获取各种信息。这种方式常常会导致信息获得不及时，而耽误工作。本方案中提出使用 Live Communications Server 2005作为统一消息系统的底层平台，客户端统一部署 Windows Messenger 5.1或Office Communicator 2005。
1、用户之间可使用上述任一种客户端进行即时通讯，可互相发送信息或文件。
2、管理员可以在一个统一的界面下查看用户的在线状况，可以对当前在线的用户发送通知消息。发送后，在线用户的客户端弹出窗口，显示通知内容。在管理员的管理界面中，可以查看已发送的消息历史记录。
3、此发送通知的服务提供通知WebService,供第三方应用系统调用。第三方应用系统可通过此接口将动态信息以消息形式发送到用户的客户端。
4、在客户端中可通过定制标签方式集成其他相关应用系统，用户则可在客户端上直接浏览其他系统中的一些定制信息，比如最新的通知通告，或者可直接对其他应用系统直接进行操作。
第三章 系统优势
3.1 技术平台优势
完全基于IE的WEB应用，迎合朝阳区政务办公平台集中部署，分散管理维护的需求，同时极大地提高了系统部署维护效率：
领航EIP系统是完全基于WEB的应用系统，与传统客户/服务器（C/S）系统比较，在对全区的统一部署、维护、升级方面提供了巨大的便利。同时也支持远程办公。
3.1.1先进的技术架构，确保系统的稳定性、扩展性等核心性能：
本系统完全基于微软.Net架构，是国际最先进的技术平台与工业标准之一，具有国际领先水平。
基于win2003的AD管理系统：对于全系统的整体人事结构“域树”的管理，可以在为人员权限管理、认证、软件自动分发、客户端群组管理方面提供巨大优势。极大提高整体系统维护水平，降低维护成本
与联机数据处理（OLAP）和数据挖掘技术(DM)的无缝挂接：在未来进行业务数据整合、分析、辅助角色时，节省大量系统迁移成本
大量第三方软件公司提供高效低价（乃至免费共享）的WebParts功能组件：作为国际标准知识管理平台，目前已提供100余种包括信息检索、项目管理、财务报表、统计分析等webparts工具软件，而且在不断增长。国内如用友软件公司等知名软件公司也会在最近推出基于该平台的版本。
3.1.2前瞻性的系统集成平台，保证企业基于国际工业标准有计划地进行信息化建设：
企业信息化是建设是一套统一规划下的分步骤实施的过程，如果没有一个面向未来、面向国际的信息平台，各个业务应用系统（如人事管理、财务管理等其他业系统）就会形成“信息孤岛”，难以整合在一起。本系统提供了企业应用集成平台，降低独立业务系统之间的应用整合投资。
本系统的体系架构，是国际上占主流地位的企业信息化架构，大多数知名软件及系统集成商都会根据该标准开发软件及应用，这对朝阳区政府各委办局之间业务对接，引入国际先进的应用系统奠定了基础；
本系统不是一个单纯的软件产品，它还提供了企业内部未来知识管理、业务系统集成、第三方系统集成的核心平台
3.1.3无需编码的个性化应用改造，使信息系统能够按企业管理流程、方法的不断变化“柔性”变动
由于各使用单位的需求千差万别，所以对表单生成、公文流程等应用的无编码可视化定制，将起到“不变应万变”的作用。
3.1.4后台系统与桌面IE浏览器、Office的无缝联接，提高系统易用性
由于企业EIP系统的使用者是企业内部全体知识员工，包括非计算机技术人员。所以整套系统的易用性，即被员工的接受程度，将是系统成败的重要因素。作为主要使用微软企业服务器产品的解决方案，领航EIP系统与微软前台浏览器和办公软件可以进行无缝连接，从而降低了普通员工对系统的掌握难度，提高了整个系统使用效率。
3.1.5 强大的“随需应变”自定义能力
管理信息系统，需要基于用户组织结构、管理策略和管理思想，提供相应的信息化执行工具。为达到这个“个性化”的目标，传统的信息化系统需要进行代码级定制开发。一旦企业内部结构或管理策略发生调整和变化，则根据原有需求定制开发的系统将无法适用。其弊端小则浪费了信息化的前期投入，大则降低了企业管理制度的实施力度。而在LHEIP.Net门户平台中（如下图所示），包括列表、应用和站点在内的任意环节，都可以随时根据用户的需求进行“无编码” 更新和重定义，真正从平台的角度诠释了企业信息化系统的结构和搭建过程。企业用户不仅可以利用LHEIP.Net门户平台“随需应变”地定制自己的信息化应用，而更可以根据企业内随时调整的管理制度“随需应变”地更新系统。
3.1.6 全线集成微软服务器产品，保证系统性能
Microsoft 系统体系结构（MSA）是一种以 Windows 为核心的技术体系结构，在合作实验室环境中经过严格的测试和验证，可提供卓越的规划和实施指南。其根据 MSA 体系结构原则和设计目标对综合的企业情境进行实际扩建，从而大大超越了传统的技术白皮书。最终得到了一个可信赖的平台，通过遵循完善的体系结构原则，全面实现了可用性、安全性、可伸缩性、可管理性和可靠性等设计目标。

3.2.1 领航科技在本项目技术领域的专业开发背景
由于朝阳区政府是微软中国在中国地区的政府合作伙伴，所以微软在相应的软件和技术服务上给予朝阳区充分的支持；同时区内政府办事人员也习惯 office等产品的使用，所以也易于接受微软平台（.net）内部办公平台。所以在培训维护方面的“使用成本”降低；微软后台产品以易用性、易维护著称于世，同样水平的微软网管的人力成本远低于UNIX网管成本，所以微软平台“维护成本”较低；由于微软在办公领域还有专门的网站管理视频会议、项目管理、客户关系、B_C电子商务、B_B电子商务等诸多全套应用服务器产品，当政府需要向上述领域扩展功能时，也可以首选这些功能实现“无缝连接”，所以功能扩展成本较低；同时，选用微软平台的办公系统，当微软相关产品升级时，还能享受相应的功能升级，所以升级成本较小。综上所述，整体拥有成本较其他平台有明显优势
3.2.2 信息办内网功能是朝阳内网的“试验田”
领航在信息办内网建设中充分发挥了自己的优势。在完成办里提出的针对性的个性工作要求前提下，积极引进微软国际标准的服务及应用。通过领航科技深层次的软件开发，配合以往众多行业的应用经验，成功地在朝阳区信息办应用并实施了专业的网站内容管理模块，即时通讯模块、引进网站群管理理念，同时实现信息办内部的办公平台应用单点登陆。
首先，在网站内容管理部分我们集中实现了网站模版的灵活自定义，栏目的灵活自定义，发布权限的自定义，让整个信息办的内部办公系统能够保持常变常新，贴近信息办业务流程的实际。
其次，在保持信息的实时更新的前提下，领航科技发挥了即时消息模块的管理功能，让系统产生或同事安排的信息，新闻，任务，流程等都主动的推送到使用者的桌面，充分显示了系统的易用性；同时可以在线显示人员登陆情况，让全体人员知道每个的工作状态。可以随时网络安排在线会议，实施发送个人邮件等。
再次，在上面模块应用的基础上，领航科技积极引进站点群管理的理念。通过建立各个应用站点，最大程度的实现了信息办内部的网络办公。即达到了信息的共享，又满足了信息办内部人员个性化的要求。
最后，领航科技充分考虑信息办将来系统扩展及其他业务系统的应用，整体设计采用微软的活动目录身份认证技术，为实现整体系统单点登陆模式奠定基础。
综上所诉，领航科技在建立信息办内部办公网系统之初就考虑了诸多因素，充分按照国际标准，为建立信息办，乃至朝阳区网络系统的标准做了坚实的准备。

领航科技从第一次与信息办合作，为用户做第一套办公系统开始就一直遵循用户至上，一切以人为本的原则。充分听取客户意见和建议，在全面进行系统需求调研的前提下，虚心进行系统应用开发。在用户初步满意的条件下，没有原地踏步，而是基于用户使用习惯的角度，基于一个专业的软件服务商的行业意识，不断完善系统，改进功能。历经3版，在办里各方面领导的积极配合下，最终完成系统的优良验收。系统不仅得到了办里领导的表扬，同时也获得区领导的认可，目前已经在朝阳区其他委办局成功进行推广应用。良好的用户口碑，优质的系统应用，周到的软件服务为我们进一步与区内，市内政府单位合作提供了有力的条件。
