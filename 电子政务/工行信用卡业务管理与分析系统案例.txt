工行信用卡业务管理与分析系统案例
项目概况：
信用卡业务实行统一管理、集中核算、分级经营的体制。总行设信用卡业务部，在省、自治区、直辖市、计划单列市分行设信用卡业务部，兼具管理和经营职能，地、市分(支)行设信用卡业务部，经办本部门信用卡业务，县级支行经办上级行委托办理的信用卡业务。
信用卡业务的基本内容包括：市场推广，发卡，持卡人服务，特约商户服务，取现网点管理，信用控制和风险管理，会计核算等。
随着信用卡发行量的增多、交易额的增大、市场的拓展以及竞争的日益激烈，不论是市场开拓人员还是管理决策人员都迫切地希望建立起一套全行信用卡业务管理和分析系统，以便安全高效地开展信用卡的各项业务。
信用卡业务管理与分析系统采用数据仓库和数据挖掘技术，为银行卡业务的管理人员和业务人员提供了客户服务分析、市场分析、经营管理分析和风险控制等诸多管理和分析功能，是行之有效的银行卡业务决策支持系统。
方案设计：
1、需求分析：
上海工行信用卡业务部设有一名总经理和若干名副总经理，下辖如下组织机构：
信控科
市场科
营业部
客户服务部
会计科
核算科
业管和事后监督科
办公室
科技科
上海工行信用卡业务部以提高经营效益、控制经营风险为目标，打算建设《信用卡业务管理与分析系统》，进一步提高管理决策的科学性。该系统为信用卡业务人员和管理人员提供业务处理、业务管理、业务分析和预测等多方面的功能，旨在为信用卡业务中的市场开拓、客户服务、风险控制等业务提供强有力的支持。
2、工程目标：
该系统含有如下几大功能：
风险管理(信控部)
个人综合消费信贷(信控部)
商户管理(市场部)
统计分析(市场部)
客户关系管理(客户关系部)
内部业务管理(会计、核算、业务管理和事后监督)
日常业务(营业部)
信息服务(办公室)
系统管理
数据管理
3、设计方案：
3.1  硬件设备
硬件设备包括一台IBM  RS/6000  F50  小型机，用作数据仓库的服务器，一台高档PC服务器，用作WEB  服务器和数据仓库的控制台，这两台机器是信用卡业务管理与分析系统的中心服务器，它们和各个科室的工作站通过局域网连接在一起。
3.2  支撑软件和应用软件
支撑软件包括IBM  HTTP  Server、IBM  WebSphere  Application  Server  3.0、IBM  公司的Visual  Warehouse  软件包(含DB2  数据库、OLAP  Server  、前端展现工具)。
《信用卡业务管理与分析系统》采用浏览器/Web  服务器/数据库服务器(Browser/Web  Server/Database  Server)软件结构。
数据管理包括数据的备份和恢复、与其他系统接口数据的倒入倒出(如黑名单),采用传统的客户/服务器(Client/Server)方式，
3.3  数据接口
《信用卡业务管理与分析系统》的数据接口包括第一次的数据初始化和每天的日常数据倒入两种情况。
3.3.1  数据初始化
数据初始化，需要个人档案、单位档案、帐户余额文件、卡信息文件、储蓄所档案、特约商户档案、POS档案、ATM档案、历史流水帐。
3.3.2  每月倒入的数据
帐户余额文件
止付文件
3.3.3  日常数据
每天需要获取的数据包括：
牡丹卡交易流水
浦江卡交易流水
磁卡POS交易流水
本代它POS交易
它代本POS交易
IC卡本代本/它交易
IC卡它代本交易流水
它代本ATM交易
本代它ATM交易
打卡文件
4、系统运行效果：
项目一期已完成，其中包括风险管理、个人综合消费信贷、商户管理、统计分析、系统管理和数据管理模块。风险管理实现了办卡的电脑化，对办卡人和持卡人的信用进行评估，对透支客户的台帐建立。个人综合消费信贷是新的业务，系统通过贷前、贷中和贷后管理，系统完全控制整个贷款流程，信贷人员可以方便的查询每个申请处于某个阶段，大大降低贷款的风险。商户管理把原有的商户档案和POS档案迁移到新的系统，在功能和数据一致性上大大增强。统计分析通过多维分析工具，实现了个人持卡人构成分析，单位持卡人构成分析，持卡人交易分析，特约商户交易分析，跨行交易分析，持卡人透支分析，特约商户构成分析等分析主题。
系统符合卡部工作的需要，得到了用户的一致好评。
