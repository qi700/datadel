电子政务应“抓住”SaaS趋势
2003至2004 年，全球在SaaS(软件即服务)上的支出增长了39%，Netcraft预测，未来12个月内的活动站点增长率将达到28%，Web服务增长率将达到19%;IDC预测，2009年，全球在SaaS上的支出将达到107 亿美元;Gartner预测，到 2009 年，美国托管收入将增长13%——这些都表明，SaaS的应用前景确实美好。
SaaS趋势在中国也正在展现，虽然目前我对中国市场的认识还不是很透彻，但是我看到，在公共市场和消费市场这种趋势正在增长，比如部分政府部门和盛大、新网互联等在线服务公司，都在采用托管方式并从中获得收益。
其中，盛大选择了Windows Server System和.NET来实现主要业务支持系统，公司内核、统一门户、交互式多媒体门户等项目和游戏服务器平台、流媒体DRM，都是基于Windows Server System实现的。现在，盛大可以每天24小时为同时在线的数百万个用户提供高性能服务;整个系统的总体拥有成本降低了30%，总体开发周期缩短了1/3;通过Microsoft的业务支持模型和合作伙伴体系，降低了业务进行连续性工作的成本。
新网互联托管平台也是基于Windows Server system 和.Net，该平台提高了系统的性能，加强了可扩展性，总体拥有成本降低40%，同时项目投放的周期也缩短了。我相信在未来，这一托管服务方式将会被越来越多的政府和企业所接受。
那么，托管的重点包括哪些内容呢?我认为，首先是Web托管，第二类是电子邮件和通信，第三类是应用程序托管。
至于为什么要与Microsoft合作?回答这个问题非常简单。微软是拥有各类最终客户所需要产品的唯一公司;微软对服务战略的重视程度非常高，并拥有多种托管产品和解决方案;微软用于托管的工具包括Visual Studio和SQL Server2005，其中Visual Studio拥有500万次以上的下载量，主要用于部署FTP 的支持工作;而用SQL Server Management Studio的VS集成功能，可以轻松实现管理的简化。Windows提供了一个完整的端到端的托管平台，应用该平台，用户可以迅速降低总体拥有成本，并获得最大的投资回报。
在这里我特别要介绍Internet Information Services (IIS) 7.0，它不只是企业级Web服务器，还是一个可扩展的平台，用来安全地交付Web上的业务应用程序和服务。IIS 7.0增强的功能包括可扩展的模块化体系结构、集成的应用程序堆栈、分布式的配置模型、委托管理工具、全面的诊断支持等，这样就为用户带来了引人瞩目的自定义解决方案，优化的安全性和修补，可伸缩的改进基础结构，快速解决方案的部署等。
最后我想指出的是，微软长期致力于托管解决方案，Windows 提供了一个完整的端到端托管平台，Windows共享托管将帮助每一个用户提高收入，降低总体拥有成本。
