浅谈国土资源网站的规范化建设
引  言
近两年来，各级国土资源管理部门相继建立起网站，除台港澳外，互联网上省级国土资源网站已达 23个。这些网站面向社会大众及国土资源行业，发布法律法规、大政方针，以及国土资源规划、管理、保护与合理利用情况等权威数据，提供了大量基础性和公益性信息，极大地推进了国土资源信息化进程。为高效率、高质量地实施“网站工程”，有哪些原则可循，有什么规范可供借鉴？本文从信息组织和网站设计两方面谈点个人看法。
一、信息组织
1.1 数据管理
信息工程理论指出，数据位于现代数据处理系统的中心，数据是稳定的，处理是多变的。与其他信息系统一样，网站的核心也是数据，政府网站首先应提供真实权威的信息。
国土资源管理部门的信息十分丰富，可以分为空间数据与非空间数据（公文、报告、论文、录音和影像资料、连续出版物、缩微胶片等）两大类。它们有的来自于办公自动化等各种应用系统，有些是独立的电子文档。为有效地利用这些数字资源，在将其发布到网站之前，先要进行信息组织——数据整理与序化。
在信息组织技术中，如何建立信息资源管理标准，简明、科学地进行实体关系分析和建立规范化的数据结构，是最基础的工作。标准必须是从实际出发的、有生命力的、切实可行的。要保持标准的简明性，标准中复杂难懂的东西越少，就越好执行。同时还要注意，任何原则都有例外的情况，没有适用于所有情况的标准，但决不能把例外当成常规。
1.2 元数据
元数据被称为是关于数据的数据，专门用于描述数据的特征和属性，可用于分布式数据发现和检索。在国土资源网站的数据管理中，对于空间数据可以借鉴数字化地理元数据内容规范（ FGDC ），而非空间数据则可以利用都柏林核心元数据（ Dublin Core ）。
（ l ）数字化地理元数据内容规范
FGDC / CSDGM （数字化地理元数据内容规范）是美国联邦地理数据委员会在 1992 年建立的地理数据文件的术语和定义集合。 1994 年通过了数字化地理元数据的内容标准，名为 CSDGM ，但通常仍叫做 FGDC 。 FGDC 具有 300 多个元素，包括 7个主要段和 3个辅助段，内含 300 多个数据项，涉及标识信息、数据质量、地理数据组织方式、地理参考信息和资料类型等。 1995 年 FGDC 成为美国国家标准，广泛用于美国国家空间数据体系。
目前我国已参照 ISO 15046-15地理信息——元数据（ CD 2.0 ）及 FGDC v.2.0 拟定了国家基础地理信息系统（ NFGIS ）元数据标准草案，提供 NFGIS 元数据的内容，包括 NFGIS 数据的标识、内容、质量、状况及其他有关特征。这一标准可用于对 NFGIS 数据集的全面描述、数据集编目及信息交换网络服务。
（ 2 ）都柏林核心元数据
国土资源网站有相当多的信息完全来自于本级政府部门，信息的生产、管理、发布与维护相对集中。这时利用元数据对数据对象进行描述，即标识出一份电子文档的主题、内容、形式特征等属性（类似图书文献编目），可以识别和评价信息资源，并追踪其在使用过程中的变化，从而能够方便地查找并组织信息资源，简单高效地管理数据对象。
Dublin Core （都柏林核心元数据， DC ）是 OCLC（联机图书馆中心）和 NCSA （美国国家超级计算机应用中心）于 1995 年 3 月联合发起，由计算机专家、网络专家和图书馆专家等人员所组成的非正式小组开发的。其目的是建立一个广泛的元数据元素集，可以描述任何网络信息资源，并足够简单以至任何作者无需专门培训就可以创建自己文件的元数据。与专业编目人员使用的传统元数据格式 MARC 不同， DC 的创建和维护十分简单，句法易于理解，并具有系统互用性和可扩展性等特点。
目前 DC 已经拥有15个基本元素，并可以使用 Type 和 Scheme 限定词以及Link 参照对元素进行扩展。依据其所描述内容的类别和范围，可分为三组：
A．对资源内容的描述： Title （资源名）、 Subject （主题和关键词）、 Description （说明）、 Source （来源）、Language（语种）、 Relation （关联）、 Coverage （覆盖范围）
B．对知识产权的描述： Creator （创建者）、 Publisher （出版者）、 Contributors （其他责任者）、 Rights （权限）
C．对外部属性的描述： Date （日期）、 Type （类型）、 Format （格式）、 Identifier （资源标识符）。

二、网站设计
2.1 栏目
网站所管理的各种数据最终要发布上网，才能最终实现网站与信息的价值。为了形成用户在浏览器中看到的各级页面，首先应根据网站的主题设置相应的栏目，然后通过自动提取属性、检索或手工选择的方法，将后台管理系统中的数据表现在页面中。
栏目设置要简单、清晰。栏目与子栏目的树型结构应保持“广度优先”，即一个栏目节点的扇出要尽量大，而层次应尽量少。
省级国土资源网站的内容基本划分四大板块：部门简介、政务动态、政策文件、政务大厅。其中部门简介主要包括组织结构、职能介绍；政务动态包括通知、新闻、时事专题；政策文件包括公告与法律法规；政务大厅包括办事程序、表格下载、办文进度查询、电子信访。
这种栏目设置模式既适应政府部门的职能与业务布署，又具有良好的可扩展性。例如，政务动态中可以开辟网上调查与在线论坛；随着信息化进程的推进，政务大厅中可以加人专题数据库，甚至实际运行资金转移、公文传送等网上办公业务。
栏目确定后，要与后台数据衔接起来，才能用丰富的信息充实栏目。通过“文献编目”，已获得了对信息资源的描述记录——元数据，它是后台管理系统的基本构成单元。而将后台数据表现在客户端页面中，主要方法即是利用元数据（如主题、作者、类型和日期等），判断信息资源的使用价值，检索出所需要的信息。具体方式有三种：
A．自动提取属性
例如提取Subject （主题和关键词）为“湿地”的所有数据（ Subject=湿地），动态生成一个有关湿地的专栏。
B．检索
可通过空间元数据搜索引擎得到一些查询结果，如全国空间数据资源的元数据列表。或根据某一查询条件（ Title = XXX and Creator = XXX ） ，组成某位专家论述某一问题的专栏。还可以直接向用户提供查询功能，如在首页上设置查询框。
C．手工选择
例如在后台管理系统中检索出所需的数据，将其具体图文内容编写成发布页面，或提供信息资源的静态链接列表。
2.2 链接结构
链接结构指页面之间相互链接的拓扑结构。设计链接结构是网页制作中的重要环节，符合逻辑的网站链接结构不仅利于浏览者的访问，也有利于网站的管理与维护。
网站的链接结构有树状和星状两种基本方式。树状链接结构即从首页分出多个分支，再由分支节点得到下一级页面，上下级双向连通。星状链接结构指每个页面相互之间都建立链接。
在实际的网站设计中，总是将这两种结构混合起来使用：首页和一级页面之间用星状链接结构，一级和二级页面之间用树状链接结构，如图所示。超过三级则建议在页面里显示导航条，帮助浏览者明确所处位置，例如：“您现在的位置是：首页一＞政务动态一＞人大政协十届一次会议专题一＞全国政协委员访谈”。
2.3 网页
网页设计即是根据网站主题和信息内容，对所必需的各种构成要素（标志、文字、图像、符号、色彩等造型要素及网站名称、标题、标语、菜单、正文等内容要素）进行视觉的关联和配置，使这些要素和谐地出现在一张网页中，创造统一的视觉效果，同时更好地表现与传达信息，引导用户正确有效地浏览。
（ l ）限制因素
在规划每个页面时，时间和空间是两个重要的限制条件，应使其达到最佳平衡状态。在空间上，访问者面对的屏幕是一个很有限的视觉空间，页面水平方向应尽量不使用滚动条，长度宜在两屏左右。在时间上，如页面使用图像、声音和视频文件，则十分耗时，会使用户失去等待的耐心。应尽量不使用声音和视频文件，图像文件也要在保证质量的前提下尽量压缩。
（ 2 ）统一网页风格
一个网站的网页风格应该具有统一性，这样能够突出网站要表达的主题，同时也能够帮助浏览者快速了解网站结构和浏览机制。可以利用模板快速批量制作具有相同或相似风格的网页。网页设计风格统一化的标准包括：
A．使用相同的版式。在各级页面上重复出现某些对象，如网站名称、标志和标语。各页面应用网站的标准字体和标准色彩，使各页面有相似的外观。如可以在每个网页的左上角放置网站标志，在右下角放置网站管理员的联系地址；
B．使用一致的导航条。在网页顶端设置导航条，引导浏览者到达感兴趣的页面；
C．根据网站主题，确定整个网站的主色调。政府网站应庄重大方，不宜使用过于强烈的色彩；
D．使用相同风格的图标。尽量使用相同的图像代表同一含义（如 New 、Hot等图标）。使用统一的图片处理效果，如阴影效果的方向，厚度，模糊度均保持一致。
2.4 首页
浏览者会根据网站的首页判断网站的价值，决定有无必要继续访问，因此，首页从形式到内容都要紧密围绕网站主题，起到对网站进行整体介绍的作用。
（ l ）将重要的内容放在首页。不要在首页放置与主题无关的内容。及时更新的内容应放在显著位置；
（ 2 ）首页应有很清楚的浏览机制，提供栏目目录／导航菜单及搜索功能，设计上要符合通常的浏览习惯，使浏览者能够很快找到需要的主题；
（ 3 ）首页应突出网站形象，包括标志、标准色彩、标准字体，吸引浏览者的注意力；
（ 4 ）首页页面大小的设置应注意到屏幕特性和用户浏览习惯。一般来说，页面水平宽度不要超过屏幕宽度（页面标准按 800 * 600 分辨率制作，实际尺寸为 778 * 434px ），长度宜在两屏左右；
（ 5 ）首页的设计应遵循简洁的原则。在版式上可借助报纸的做法，如划分版块、分栏。页面上所有元素都应当有明确的含义和用途，不要放置无用的小摆设；
（ 6 ）与浏览者交流的内容，应体现在首页上。如在首页放置网上投票、网上调查等功能。
2.5 编码
在网站的目录设置，文件命名以及 Java Script 或 ASP 书写等方面遵循统一的格式，可以使程序员保持一致的编码风格，不仅能够提高代码的可读性，更利于网站的管理与维护。
